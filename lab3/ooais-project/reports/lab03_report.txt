Consider the following:
- Why should the target column not be included in model input? Because if the model has access to the "answer" during training, it will simply learn to map the input directly to that column instead of identifying patterns in the features (data leakage). This results in a model that appears perfect during training but fails completely when predicting on new, unseen data.
- What could happen if irrelevant columns were included? It introduces noise into the system. Machine Learning models might identify false patterns where none exist. This increases the complexity of the model, slows down the training process, and typically leads to overfitting, where the model performs poorly on real-world data because it focused on irrelevant details.
- Should feature selection always be performed in the ingestion layer, or later in the pipeline? While basic cleaning and filtering (removing obvious identifiers like IDs) happens in the ingestion layer to save memory and storage, advanced feature selection is performed in later stages use statistical methods (like correlation analysis) to decide which specific features are more useful for the model.

P2: Pipeline Thinking

Explain:
- why ingestion must precede preprocessing? Because it establishes the structural integrity of the data (checking columns and formats) before any transformation occurs. Without a successful ingestion, preprocessing would attempt to operate on non-existent or corrupted data structures.
- what happens if invalid data is not removed? Numerical calculations will crash the system. Furthermore, these records introduce noise that prevents the model from learning the actual physical patterns of orbital observations.
- how ingestion affects model quality? By filtering out corrupted records and selecting only relevant features, it ensures the model is trained on "clean" data. High-quality input leads to higher predictive accuracy and more reliable anomaly detection.
